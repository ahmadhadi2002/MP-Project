Rainbow Table Generation

Although rainbow table and dictionary cracking may be similar, they differentiate in how they handle storing of information. For dictionary cracking, a set of characters are stored, hashes could also be stored in this manner but would get very large very quickly. An example of this, a plaintext and a 32 Byte Hash. With a library of 1 Billion passwords would be 40GB of data that a computer would need to sort through, which can take up a lot of time and computational power.

Instead rainbow tables uses a reduction function to generate the table. In a rainbow table contains many different “chains” with the starting of each “chain” being a plaintext password. The rainbow table will only store the plaintext of the first plaintext value of a “Chain”.

This chain would then be hashed and the first couple of characters will be used to generate a new hash text. If a hash matches the password hash, it will then look back at the plaintext value stored within the rainbow table. From there it will go down the “Chain” and look for the plaintext value of the matching hash. The exercise below will allow you to try a rainbow table generation.
